# 📊 项目分析：Agent Platform

## 一、项目概览

这是一个 **AI Agent 智能助手平台**，基于 `Python + FastAPI + LangChain + LangGraph + Milvus + Redis + PostgreSQL` 技术栈构建。支持两种 Agent 运行模式：**ReAct 推理循环** 和 **RAG 检索增强生成**。

| 维度 | 详情 |
|------|------|
| 语言/框架 | Python 3.12+ / FastAPI / Gradio |
| LLM | DeepSeek (deepseek-v4-pro)，OpenAI 兼容接口 |
| 嵌入模型 | 阿里云 DashScope text-embedding-v4 (1024维) |
| 向量库 | Milvus 2.5.0 (Standalone) |
| 缓存 | Redis 7 |
| 数据库 | PostgreSQL (asyncpg) |
| 编排 | LangGraph StateGraph (ReAct) + LangChain LCEL (RAG) |
| 可观测性 | 控制台 / OpenTelemetry → Jaeger |
| 基础设施 | Docker Compose 一键部署 (PG + Redis + etcd + MinIO + Milvus + Jaeger) |

## 三、核心模块详解

### 3.1 Agent 引擎 ([app/agent/](app/agent/))

| 模块 | 文件 | 说明 |
|------|------|------|
| ReActAgent | [react_agent.py](app/agent/react_agent.py) | LangGraph StateGraph 驱动 思考→行动→观察 循环，支持工具调用，最大 10 轮迭代 |
| RAGAgent | [rag_agent.py](app/agent/rag_agent.py) | LCEL 管道：检索→拼 Prompt→LLM 生成，temperature=0.3 抑制幻觉 |
| AgentExecutor | [executor.py](app/agent/executor.py) | 根据 `AgentRequest.mode` 路由分发 |

### 3.2 三层失败防护

> 详见 [目前关于agent的优化.md](目前关于agent的优化.md)

```
CircuitBreaker (工具级) → consecutive_failures (轮次级) → 兜底话术 (用户级)
```

这是项目最亮眼的设计——每层独立工作、逐级兜底。

### 3.3 RAG 管道 ([app/rag/](app/rag/))

| 模块 | 文件 | 说明 |
|------|------|------|
| 文档加载 | [document_loader.py](app/rag/document_loader.py) | 支持 .txt / .md / .docx / .pdf / .pptx 五种格式 |
| 文本分块 | [chunker.py](app/rag/chunker.py) | 中文感知递归分块，`。` 作为高优先级分隔符 |
| 嵌入工厂 | [embedder.py](app/rag/embedder.py) | 可插拔工厂模式，支持 dashscope / openai_compatible / sentence_transformers |
| 向量检索 | [retriever.py](app/rag/retriever.py) | Milvus ANN 检索，IVF_FLAT 索引，内积相似度 |
| 重排序 | [reranker.py](app/rag/reranker.py) | 分数过滤 → 去重 → 截断，min_score=0.3, max_docs=3 |

### 3.4 工具体系（三层架构）

| 层级 | 说明 | 示例 |
|------|------|------|
| 原生 Tool | 单次调用、无状态、原子操作 | 安全计算器 (AST白名单)、知识库搜索 |
| Skill | 多步编排、有内部流程 | 财报分析 ([financial_report.py](app/skills/financial_report.py)) |
| MCP 远程工具 | 运行时通过 JSON-RPC 动态发现 | [mcp_client.py](app/tools/mcp_client.py) / [mcp_tools.py](app/tools/mcp_tools.py) |

## 四、亮点设计

| 亮点 | 说明 |
|------|------|
| **三层失败防护** | 熔断器 → 连续失败计数 → 兜底话术，防止 LLM 死循环 |
| **AST 安全计算器** | 白名单节点求值，杜绝 `eval()` 代码注入 |
| **中文感知分块** | `。` 作为高优先级分隔符，保证 chunk 语义完整 |
| **可插拔嵌入工厂** | 改 `.env` 一个变量即可切换嵌入模型 |
| **可插拔追踪** | console / OpenTelemetry 一键切换 |
| **故障类型区分** | LLM 能区分 `[熔断]`、`[工具异常]`、`[未知工具]`，做出不同决策 |
